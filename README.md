1. How do I approach designing software?

I try to understand the problem and break it down visually before writing anything. This project further pushed that thinking, looking at a real object and figuring out what simple shapes make it up before touching code made the whole process much smoother. It's an approach I'll keep using.

2. How do I approach developing programs?
   
I build things in stages and treat mistakes as information to build off of. Starting with plain colored geometry before adding textures saved me a lot of debugging time, and the trial and error involved in getting things like the DS lid rotation right taught me to be patient with iteration.

4. How can computer science help me in reaching my goals?
   
Getting hands-on with the OpenGL pipeline made concepts like matrix transformations and UV mapping click in a way that reading about them never did. That kind of practical understanding, and the problem-solving mindset that comes with it, is something I'll carry into both future coursework and my career.
